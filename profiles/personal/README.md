# Personal Profile

Primary fit:
- Pi for personal use

## Intent

Optimize for speed, low friction, and local experimentation on the author's own machines.

## Expected defaults

- Runtime: Pi-first
- Providers / models: personal/local preference is acceptable; optimize for speed and iteration, within the repo's existing safety defaults
- Environment: local workstation, tmux, worktrees, repo-linked dotfiles
- Allowed commands / tools: Pi `/skill:*`, `/scout`, `/worker`, `/reviewer`, `/handoff*`, repo worktree helpers, and local shell tools are all normal in this profile
- Autonomy: higher, within local safety rules
- Extensions / integrations: repo-tracked Pi extensions and personal local tools are acceptable
- Safety posture: local-first; keep risk bounded to the machine and repo, avoid unnecessary hidden automation, still review risky changes before considering them done

## Practical differences

- Faster iteration is acceptable when blast radius stays local.
- Local convenience integrations are acceptable if they remain understandable and reversible.
- Personal context can rely on repo conventions plus local shell/env setup.
- Profile selection is manual for now; use this profile when the task is personal/local and does not need work-facing constraints.

## Non-goals

- This does not define a separate workflow.
- This does not imply bypassing review for risky changes.
- This does not require separate copies of shared docs or templates.
